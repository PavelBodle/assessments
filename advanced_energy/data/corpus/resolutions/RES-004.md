# Resolution Note — GlobalConnect VPN unstable since v5.2 update

## Summary
Ticket TCK-00038 (P3) reported by USR-00118 in Marketing: "GlobalConnect VPN unstable since v5.2 update".
VPN was fine last week. Now it disconnects within minutes of connecting. Reinstalling did not help.

## Affected Scope & Environment
Affected user USR-00118 (Marketing); system/asset: VPN.
Related/duplicate tickets exist for this recurring problem.

## Diagnosis & Root Cause
A regression in VPN client v5.2 set the dead-peer-detection timeout too low, dropping tunnels on brief latency spikes common on home Wi-Fi.

## Resolution / Recommended Fix
Roll back to VPN client v5.1.3 or apply hotfix v5.2.1 which restores the DPD timeout to 30s. Reconnect over a wired link while patching.

## Verification
Confirmed with the user that the issue no longer reproduces after the fix; monitored for 24h with no recurrence.

## Preventive Action / KB Update
Add a proactive check and user guidance to prevent recurrence. See KB-001 for the step-by-step guide.
